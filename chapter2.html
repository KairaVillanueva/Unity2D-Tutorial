Programming Terms

Conditional Statements

Making a tough decision requires us to momentarily pause and think about which options we have available. Only when we have finally concluded on which choice we want to pursue would we then follow it with a set of actions. Conditional statements function in a similar way. The programmer determines what type of environment they want, so they propose a condition that either evaluates to true or false. To relate this to a more familiar concept, let’s look at passwords. When a new username is created, a password is also asked to be inputted. The program stores the password, and in order to successfully log-in, the user must type the exact password designated to the username.


In this game, we will use a conditional statement to check if the bullet has hit the asteroid. What do you think will happen after we set that conditional statement? Of course, the asteroid will be destroyed and we could also add a point for the player (or deduct a point if you’re feeling devious).


If (object collides with asteroid)
    Destroy the asteroid
    Score + 1
